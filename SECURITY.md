# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the UseSense Android SDK, please report it responsibly.

**Do not** open a public GitHub issue for security vulnerabilities.

### How to Report

Email **security@usesense.ai** with:

1. A description of the vulnerability
2. Steps to reproduce
3. Potential impact assessment
4. Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: Within 2 business days of your report.
- **Initial Assessment**: Within 5 business days, we will provide an initial severity assessment.
- **Resolution**: We aim to resolve critical vulnerabilities within 14 days. Non-critical issues are addressed in the next scheduled release.
- **Disclosure**: We will coordinate disclosure timing with you. We request a 90-day disclosure window for critical vulnerabilities.

### Scope

This policy covers:

- The UseSense Android SDK (`com.usesense.sdk`)
- SDK communication with the UseSense API
- Data handling within the SDK (frames, audio, metadata)

### Recognition

We appreciate responsible disclosure and will acknowledge security researchers in release notes (with your permission).
